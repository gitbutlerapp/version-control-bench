## Headline

359/360 canonical runs passed the deterministic verifier. Plain `git` and GitButler each passed 120/120; Jujutsu passed 119/120. The one miss was a Claude Jujutsu split-commit run with the right final files but one extra commit in the graph.

GitButler remained the fastest and least command-heavy arm overall. Against plain `git`, its mean wall time was 63.1% lower for Codex and 67.3% lower for Claude, with 77.8% and 78.6% fewer task-relevant version-control commands respectively. Jujutsu was 19.1% slower than `git` for Codex and 47.9% slower for Claude; it used 2.9% more commands for Codex and 30.4% fewer for Claude.

The task-clustered comparison supports the command-count result for both agents and the wall-time result for Codex. Claude's GitButler wall-time interval remains wide over six tasks. Claude Jujutsu's paired wall-time interval is entirely above zero; the other Jujutsu intervals cross zero.

## Run Integrity and Excluded Attempts

The published aggregate is a strict rectangle: 6 tasks × 2 agents × 3 arms × k=10 = 360 canonical rows. It was recomputed from four source batches rather than reusing their summary objects.

Fifteen infrastructure-failed attempts were preserved outside the canonical aggregate across 12 canonical slots: four HTTP 529 responses, five HTTP 500 responses, three HTTP 401 responses, and three interrupted runs. Each affected slot was rerun from a fresh isolated workspace, and every canonical replacement passed. The failures spanned `git`, `but+skill`, and `jj+skill`; none contributes to the pass rates, timing, or command counts below. Claude OAuth was refreshed and an explicit Opus probe passed before continuation.

Some preserved attempts share a canonical run ID with their replacement. The merged `manifest.tsv` and `aggregate.json`, not recursive directory scans, define the canonical set.
