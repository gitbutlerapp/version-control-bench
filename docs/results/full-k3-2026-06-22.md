## Follow-Up: Claude Squash Outlier

Date: 2026-06-23

Batch: `tmp/pilot-runs/pilot5-claude-fix-20260623-000830`

The original Claude `but+skill` squash mean was worse than `git` only because run 3 spent about a minute over-verifying after the correct squashes. The trace showed the actual `but` commands were fast: 13 task VC commands total, but only about 0.62s of observed task VC runtime. Claude saw GitButler status output with an upstream reference block and a `◐` status marker, then inspected raw commit objects to prove it was not a conflict.

Fix: add generic `but status` interpretation guidance to the generated `but+skill` arm instructions: upstream blocks above separators are reference-only, and `●` / `◐` symbols are status decorations rather than conflict evidence by themselves.

Rerun result:
